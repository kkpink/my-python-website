Importing & Using Modules and Packages

Import Variants

import math               # Imports the whole module
from math import sqrt     # Imports only sqrt
import math as m          # Imports with alias
from math import *        # Imports everything (not recommended)

Nested Modules & Packages

You can import submodules from packages:

from package.module.submodule import function

Using dir()

Lists available names in a module or scope.

import math
print(dir(math))

sys.path

Shows the list of directories where Python searches for modules.

import sys
print(sys.path)

Evaluations with math Module

import math
print(math.ceil(4.3))     # 5
print(math.floor(4.9))    # 4
print(math.trunc(4.7))    # 4
print(math.factorial(5))  # 120
print(math.hypot(3, 4))   # 5.0
print(math.sqrt(16))      # 4.0

Generating Random Values

import random
random.seed(1)                 # Set seed for reproducibility
print(random.random())         # Random float [0.0, 1.0)
print(random.choice([1, 2]))   # Random element from list
print(random.sample(range(10), 3))  # Random 3 unique items

Platform Module

Use platform to check OS and system info:

import platform
print(platform.platform())
print(platform.machine())
print(platform.processor())
print(platform.system())
print(platform.version())
print(platform.python_implementation())
print(platform.python_version_tuple())

Creating and Using Your Own Modules

Why Use Custom Modules?

  • Organize reusable code into separate files
  • Improve readability and reusability

Creating a Module

Just save a Python file (e.g., mytools.py) and import it:

# mytools.py
def greet(name):
    return f"Hello, {name}!"
# main.py
import mytools
print(mytools.greet("Alice"))

__pycache__ Directory

Stores compiled bytecode versions of modules.

The __name__ Variable

# Only runs if this file is executed directly
if __name__ == "__main__":
    print("Running as main")

Public vs. Private

  • Public: regular names
  • Private: names starting with _

Packages & __init__.py

A directory with an __init__.py file is treated as a package.

Nested Packages vs. Directory Trees

Packages can contain sub-packages in a structured directory layout.

myproject/
├── main.py
└── tools/
    ├── __init__.py
    └── stringtools.py
from tools import stringtools
