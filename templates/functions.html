Python Functions: Parameters, Arguments & Scope

Parameters vs. Arguments

  • Parameters are the variable names used in a function definition.
  • Arguments are the actual values passed to a function when it is called.

def greet(name):  # name is a parameter
    print(f"Hello, {name}!")

greet("Alice")  # "Alice" is the argument

Positional Arguments

Values are passed in order and matched to parameters by position.


def full_name(first, last):
    print(f"{first} {last}")

full_name("Jane", "Doe")

Keyword Arguments

You specify which parameter each value goes to by name.


full_name(last="Doe", first="Jane")

Mixed Arguments

You can combine both — positional first, then keyword.


full_name("Jane", last="Doe")

Default Parameter Values

You can give parameters default values in the function definition.


def greet(name="stranger"):
    print(f"Hello, {name}!")

greet()         # Hello, stranger!
greet("Alice")  # Hello, Alice!

Name Scopes & Variable Shadowing

Variables exist in different scopes: local (inside functions) and global (outside functions).

When a local variable has the same name as a global one, it shadows the global version.


x = 10  # global

def show():
    x = 5  # local, shadows global
    print(x)

show()     # Output: 5
print(x)   # Output: 10

The global Keyword

To modify a global variable inside a function, use global.


count = 0

def increase():
    global count
    count += 1

increase()
print(count)  # Output: 1

Tips

  • Use keyword arguments for clarity.
  • Default parameters go last in the function definition.
  • Avoid using global unless necessary — it's better to return values.
