Python Operators

Operators are special symbols or keywords that perform operations on variables and values to produce an outcome.

Arithmetic Operators

+   -Addition:Adds two values together       5 + 2 = 7
-   - Subtraction: Takes away one value from another   5 - 2 = 3
*   - Multiplication: Multiplies two values  5 * 2 = 10
/   - Division: Divides one value by another       5 / 2 = 2.5
//  - Floor Division:Divides one value by another and rounds the answer down  5 // 2 = 2
%   - Modulus: divides a value by another and outputs the remainder 5 % 2 = 1
**  - Exponentiation: puts one value as the power of another 5 ** 2 = 25

Comparison Operators

Compare two values and returns a boolean value:True or False 

==  Equal to: You need to use double == as a single = refers to assignment of values  5 == 5 → True
!=  - Not equal to:    5 != 3 → True
>   - Greater than:    5 > 2  → True
<   - Less than:       5 < 10 → True
>=  - Greater or equal 5 >= 5 → True
<=  - Less or equal    5 <= 4 → False-

Logical Operators

Used to combine multiple conditions.

and  Outputs True if both conditions are True
or  Outputs True if at least one condition is True
not  Reverses the  value, if the value True it outputs False and vice-versa

# Example:
x = 10
print(x > 5 and x < 20)   # True
print(not x == 10)        # False

Assignment Operators

Assign and update the value of a variable.

 =   # x = 5
+=  # x += 2  → x = x + 2
-=  # x -= 1  → x = x - 1
*=  # x *= 3  → x = x * 3
/=  # x /= 2  → x = x / 2
//= # x //= 2
%=  # x %= 3
**= # x **= 2

Bonus: Identity & Membership Operators

Used to check object identity or list membership.

# Identity Operators
is      Used to get True if same object
is not  Used to get True if not the same object

# Membership Operators
in       # True if value is in sequence
not in   # True if value is not in sequence

# Examples:
x = [1, 2, 3]
print(2 in x)         # True
print(x is [1, 2, 3]) # False (different objects)

Tips

  • Use parentheses to group complex logic: if (a > 0 and b < 5)
  • Always test boolean logic carefully
  • is checks identity, not equality — don’t confuse with ==
