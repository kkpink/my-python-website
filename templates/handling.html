Python Exception Handling

Exceptions are errors that occur while your program is running. If not handled, they can crash your program. Python gives you tools to catch and deal with these errors using try and except.

Basic Structure


try:
    # Code that might cause an error
    risky_operation()
except SomeError:
    # Code to run if an error happens
    handle_the_error()

Full Try-Except-Else-Finally

You can add else to run code if no errors happen, and finally to run code no matter what.


try:
    number = int(input("Enter a number: "))
    result = 100 / number
except ValueError:
    print("That was not a number.")
except ZeroDivisionError:
    print("You can't divide by zero!")
else:
    print(f"Result is {result}")
finally:
    print("This runs no matter what.")

Why Use Exception Handling?

  • Prevents your program from crashing
  • Lets you show helpful messages to users
  • Makes your code more reliable and professional

Common Exceptions

  • ValueError – when the value is the wrong type
  • ZeroDivisionError – dividing by zero
  • TypeError – wrong type used in operation
  • IndexError – list index out of range
  • KeyError – dictionary key doesn't exist
  • FileNotFoundError – file can't be found

Tips

  • Always catch specific errors, not just except:
  • Use finally to close files or clean up
  • Don’t hide errors — handle them clearly

Python Exception Hierarchy

All exceptions in Python are part of a class hierarchy. Understanding this helps you write better try-except blocks and catch the right kind of errors.

Top-Level Structure

  • BaseException
    • SystemExit – raised when the interpreter is exiting
    • KeyboardInterrupt – when user hits Ctrl+C
    • GeneratorExit
    • Exception
      • ArithmeticError
        • ZeroDivisionError
        • OverflowError
        • FloatingPointError
      • ValueError
      • TypeError
      • IndexError
      • KeyError
      • ImportError
      • FileNotFoundError

Why It Matters

  • Catching Exception handles most user-level errors.
  • Catching BaseException is dangerous — it may stop system-level signals too.
  • You can catch a group of errors using a parent class (e.g., catch ArithmeticError instead of just ZeroDivisionError).

Example


try:
    result = 10 / 0
except ArithmeticError:
    print("Something went wrong with math!")
except Exception:
    print("A general error occurred.")
